## What shipped this session

**Map screen (`/map`) — was completely broken, now fixed**
- Root cause was three separate, stacked issues, diagnosed one at a time via the browser Network tab and server logs rather than guessing:
  1. **Maps Static API** wasn't enabled on the Google Cloud project — it's a *separate* API from the Places/Routes APIs the rest of the app already used, needs enabling independently even on the same project/key (`console.cloud.google.com/apis/library/static-maps-backend.googleapis.com`).
  2. Billing wasn't attached to the Google Cloud project — required even to use free-tier Maps Platform APIs.
  3. `.env`'s `GOOGLE_PLACES_API_KEY` was **stale** — didn't match the key actually shown in the Cloud Console (a new key had been generated at some point and never synced to `.env`). This was the one that actually mattered; the first two were red herrings that looked plausible but weren't sufficient on their own.
- `api/static-map.js` improved to forward Google's actual error text + status code in its JSON response and log it server-side (`console.error`), instead of a bare generic 502 — this is what let us see the real "not authorized" message and root-cause it instead of guessing.
- Decided **not** to build interactive pan/zoom (would require either exposing a client-side Google Maps JS key or switching to Leaflet/OSM) — user chose to keep the static image after reviewing the tradeoff.
- `.map-image` given a `1px solid var(--border-strong)` border and corrected `border-radius` from `--radius-lg` (16px, wrong) to `--radius-sm` (8px, matches Figma nodes 197:17761 / 273:16492).

**Finalise & Save — the big one, several rounds of chip-layout debugging**
- **Interest chips "wonky" layout** — reported 4+ times before actually landing on the real cause(s). Worth reading closely if anything like this recurs:
  1. `.chip` was missing `white-space: nowrap` / `flex-shrink: 0` — long labels (e.g. "Civil Rights History") were getting squeezed and wrapping internally, distorting that chip's size and throwing off the whole row's wrap. Fixed.
  2. A wrong turn: misread Figma's `py-px` Tailwind class as literal 1px padding and built a separate, much-too-short `.category-chip` class. **Always check `get_metadata` for actual frame dimensions, don't trust hand-parsed Tailwind class names for sizing** — the real component is 36px tall, identical to the existing `.chip`. Reverted, `.category-chip` no longer exists.
  3. The actual remaining "wonky" symptom (text sitting at the top of the chip instead of centered) turned out to be that `.chip` never had explicit `display: inline-flex; align-items: center; justify-content: center`. A `<button>` (Trip Input's picker) gets free vertical-centering from the browser's UA stylesheet; a plain `<span>` (Finalise's read-only chips) does not. Same class, different tag, different rendering — fixed by making centering explicit.
  4. The "one long chip alone, two short ones together" wrapping pattern that kept recurring across different destinations is **not a bug** — confirmed via the user's own DevTools computed-styles panel that colors/radius/height all match Figma exactly, and confirmed Trip Input's `.interests-card` uses the identical `display:flex; flex-wrap:wrap` CSS already. It's inherent to flex-wrap with variable-length labels, same as GitHub topic tags or npm keyword pills. No further layout change was made or is needed here.
  5. Also added `flex: 0 0 auto; width: fit-content;` as extra defensive hardening against any stretching.
  6. Per explicit request, Finalise's chips now render **unselected/default** style (`chip` only, not `chip chip--selected`) — this is a deliberate deviation from the Figma "CategoryChips" dark-fill node (52:7859) the user originally pointed at; flagged once, not reverted.
- **`MapPinIcon`** (Destination row) replaced with the same pin+circle asset already used in `Home.jsx`'s `PinIcon` — the previous one was an outline-only pin with no inner circle.
- **Star rating**: filled star untouched; empty/unselected star replaced with the user's exact literal SVG (native 22×22, scaled to 24×24 to match the filled star's footprint), `#94A3B8` mapped to `var(--text-disabled)`.

It was floating in the vertical middle of wrapped two-line text instead of sitting next to the top line — same root-cause pattern as the chip centering bug above, worth remembering as a recurring failure mode (`align-items: center` + wrapping text = floating icon).
- **Price range formatting** (`Accommodation.jsx formatAmount`): amounts ≥ 1000 now use `Intl.NumberFormat` compact notation (`800k – 6m` instead of `800,000 – 6,000,000`), suffix lowercased. Currencies with small face values (EUR, USD) are untouched.

Fixed at the **base `.pill-button` class**, so this also protects every other pill-button in the app, not just this one row.

## Established conventions worth preserving

- **"Ask for the real SVG" rule**: when the user supplies a literal SVG, paste it verbatim rather than approximating — convert hardcoded hex to the matching CSS token where one exists exactly, otherwise leave it. When Figma serves flattened image assets instead of real vector paths, hand-approximate and say so in a code comment.
- **File deletion is blocked** in this sandbox (`rm` → "Operation not permitted"); the user explicitly declined the delete-permission prompt once. Workaround: overwrite unwanted files with deprecation-stub comments instead of leaving functional duplicate code.
- **Always verify Figma specs by fetching the actual cited node** (`get_design_context` / `get_metadata` / `get_screenshot`) rather than pattern-matching from memory or a similar-looking node seen earlier — two separate mistakes this session (the category-chip height, the delete-modal red-button assumption) came from skipping this.
- **When a bug is reported multiple times and a fix "should" have worked**, stop guessing from screenshots and ask the user to check the browser DevTools **Computed** styles panel for real ground truth. That's what actually closed out the chip saga.
- **`align-items: center` + text that can wrap onto multiple lines = a floating icon.** This exact bug showed up twice this session (Finalise's heart icon, the hotel-card checkmark) — `flex-start` is very likely the right default for any icon+text row where the text isn't guaranteed single-line.
- **`.env` changes need a full `vercel dev` restart**, not just a browser refresh — CSS/JS hot-reloads fine via Vite, env vars don't.
- **Google Maps Platform**: Places API and Maps Static API are separate APIs requiring separate enablement even under the same Cloud project/key. If a "key not authorized" error persists after enabling the API and attaching billing, check whether the key in `.env` actually matches the current key shown in the Cloud Console — keys can get regenerated without `.env` being updated to match.
